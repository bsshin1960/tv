# TV 방영 프로그램 서비스 (TV ON) - 개발 계획 & 도움말 (Plan & Help)

이 문서는 사용자의 요구사항(하단 EPG/그리드 제거 및 사이드바 통합, 전체화면 해상도 개선, 자동 재생 정책 우회, 시작 시 M3U 자동 적재, 스마트 업로드 모달 도입, 런처 창 자동 종료, 마우스 전용 화면 조작 패널 및 휠 줌)에 맞춰 고도화된 **"TV ON"** 웹 애플리케이션의 개발 내역 및 사용 설명서입니다.

---

## 1. 개발 완료 요약 & 빌드 검증

플레이어 하단의 불필요한 요소들을 완전 삭제하여 비디오 플레이어와 좌측 채널 선택 사이드바로 화면을 꽉 채운 프리미엄 IPTV 레이아웃을 완성했습니다. 또한 해상도 스케일링 오류 해결, 크롬 자동 재생 우회, 런처 CMD 자동 종료 등의 유기적인 사용자 경험 개선을 완료하였으며, 키보드가 없는 터치 환경 및 마우스 전용 환경을 위해 마우스 휠 줌 및 투명 플로팅 화면 제어 패널(첨부 박스)을 추가하였습니다.

### 빌드 결과 확인
* **빌드 명령어**: `npm run build`
* **검증 결과**: 타입스크립트 컴파일 및 Vite 번들링을 무결하게 통과하여 정적 배포 준비 완료.

## 3. 핵심 UI 조작 매뉴얼 (Help)

### A. 플레이어 하단 요소 완전 제거 & 사이드바 통합 네비게이션
* **설명**: 화면의 번잡함을 극대화하던 플레이어 하단의 EPG 일정표 및 방송 카드 그리드, M3U 로컬 제어 패널을 모두 삭제했습니다.
* **동작**:
  - 이제 방송 카테고리(지상파, 인터넷 방송, 홈쇼핑 방송, 즐겨찾기, 전체, M3U 방송)와 관계없이 화면 좌측의 **채널 리스트 사이드바가 상시 노출**됩니다.
  - 사이드바 내에 **고정 채널 검색창**이 삽입되어 대량의 채널 목록 속에서도 빠르고 스마트한 검색이 가능합니다.
  - 채널 리스트의 스크롤바가 **가장 왼쪽**에 배치되어 리스트 텍스트를 가리지 않고 원활하게 스크롤할 수 있습니다.

### B. 스마트 M3U 자동 로드 & 업로드 모달
* **시작 시 자동 재생**: 프로그램 실행 시 카테고리가 기본적으로 `M3U 방송`으로 선택되며, `public/Korea.m3u` 파일을 자동으로 읽어와 첫 번째 채널을 자동 재생합니다.
* **스마트 업로드 모달**: 헤더의 `M3U 파일 업로드` 버튼을 클릭하면 세련된 팝업 모달이 열립니다.
  - **서버 내 파일 적재**: public 폴더에 보관된 파일명(예: `Korea`, `Adult` 또는 `World 2025`)을 적고 불러오기를 누르면 즉시 다운로드하여 재생합니다.
  - **로컬 탐색기 Fallback**: 입력창을 비워두거나 서버에 없는 파일명을 입력한 경우, 즉시 PC나 휴대폰의 **로컬 파일 탐색기**를 띄워 사용자가 가지고 있는 M3U 파일을 불러올 수 있도록 안전하게 연결합니다.

### C. 꽉 찬 화면 재생 (Full Display) 및 자동 재생 우회
* **기기 맞춤형 풀 스크린**: 전체화면 버튼을 누르면 비디오 플레이어 영역만 화면에 꽉 차게 확장(Header 및 Sidebar 차단)되며, 모바일 폰이나 고해상도 4K 모니터 등 기기 사양에 맞춤형으로 찌그러짐이나 여백(Black bar) 없이 화면 전체에 확장됩니다 (`object-fit: fill` 기반).
* **ESC 키 해제 지원**: 전체화면 모드에서 키보드의 **`ESC`** 키를 누르면 원래의 사이드바/헤더 조합 화면으로 돌아옵니다.
* **자동 재생 규제 우회**: `run.bat`로 프로그램 실행 시, 크롬 브라우저에 `--autoplay-policy=no-user-gesture-required` 옵션을 부여하여 음소거 대기나 플레이 버튼 오버레이 없이 첫 화면 실행 즉시 영상과 오디오가 실시간 재생됩니다.

### D. 깔끔해진 헤더 스타일
* 불필요해진 수동 M3U 로드 토글 버튼을 지우고, 카테고리/그룹 선택 상자의 붉은색 테두리를 차분한 은회색 테두리로 단일화하여 시각적 몰입감을 대폭 높였습니다.

### E. 수평 진행바 좌우 버튼(이전/다음) 및 랜덤 재생(Shuffle)
* **수동 네비게이션**: 진행률 스크롤바의 좌측에 **이전 영상**(`SkipBack`) 버튼을, 우측에 **다음 영상**(`SkipForward`) 버튼을 동일한 수평선상에 배치하여 수동으로 간편하게 전단계/다음단계 영상으로 이동할 수 있습니다.
* **랜덤 재생(셔플)**: 스크롤바 좌측(이전 영상 버튼 옆)의 **셔플**(`Shuffle`) 버튼을 누르면 랜덤 재생이 활성화됩니다.
  - 활성화 시 버튼이 붉은색으로 강조되며, 영상 시청이 완료되거나 이전/다음으로 넘길 때 목록 중 무작위 영상을 선택해 재생합니다.

### F. 전체화면 마우스 자동 숨김 & 90% 투명 진행바
* **마우스 커서 자동 숨김**: 동영상 재생 중 마우스 움직임이 3초간 멈추면 화면에서 **마우스 커서가 자동으로 사라집니다**. 마우스를 다시 움직이브로 부드럽게 나타납니다.
* **진행바 투명도 90% 적용**: 전체화면이나 가로 모드 전체화면에서 스크롤바와 아이콘들이 영상을 가리지 않도록 **90% 투명(불투명도 0.1) 상태로 faint하게 표현**됩니다. 조작을 위해 마우스를 진행바 위로 올리면(Hover) 투명도가 25%(불투명도 0.75)로 복원되어 편리하게 조작할 수 있습니다.
* **마우스 탈출 개선**: 드래그 조작 도중 마우스가 브라우저 창 밖으로 빠르게 이탈해도 드래그가 유지되어 화면이 마우스에 달라붙는 현상을 해결하기 위해, 창을 벗어나는 즉시 드래그 상태를 안전하게 해제하도록 글로벌 예외 처리를 추가했습니다.

### G. 마우스 전용 화면 조작 패널 & 마우스 휠 줌 [신규]
키보드가 전혀 없이 오직 마우스만을 가지고 화면 스케일링, 방향, 위치 및 회전을 모두 정밀하게 조작할 수 있는 컨트롤 기능입니다.
* **마우스 휠 줌(Zoom)**: 비디오 재생 영역 위에서 마우스 휠을 굴리면 영상 비율을 깨뜨리지 않고 화면 전체를 간편하게 부드럽게 확대 및 축소(최소 0.5배 ~ 최대 3.0배)할 수 있습니다.
* **화면 제어 패널(첨부 조작 박스)**:
  - 평소에는 화면을 방해하지 않도록 **투명도 90% (불투명도 10%)**로 매우 투명하고 얇게 화면 우측에 대기합니다.
  - 사용자가 마우스 커서를 패널 근처로 가져가 접근시키면 **투명도 50% (불투명도 50%)**로 선명하게 활성화되어 조작 단추들을 선명하게 노출합니다.
  - 패널 내의 각 방향키 및 아이콘을 마우스 왼쪽 버튼으로 클릭하여 다음을 직접 제어할 수 있습니다:
    - 🔍 `전체 확대/축소` (Zoom In / Zoom Out)
    - ↔️ `좌우 너비 조절` (Stretch / Shrink Width)
    - ↕️ `상하 높이 조절` (Stretch / Shrink Height)
    - 🔄 `90도 시계방향 회전` (Rotation)
    - 🎯 `화면 비율 및 위치 초기화` (Reset)
* **PC 최대화면(전체화면) 최적화**:
  - PC 전체화면(`.player-wrapper:fullscreen`) 모드로 진입 시, 먼 거리의 대형 모니터나 TV에서도 마우스 포인터로 클릭하기 쉽도록 패널 크기와 버튼 및 아이콘 스케일이 **비례하여 자동 확대(Button: 32px ➔ 40px)**되도록 스타일 최적화를 추가했습니다.
  - 전체화면 조작 시 마우스 멈춤으로 인한 커서 자동 숨김 때문에 조작이 중단되지 않도록, **마우스가 조작 패널 위에 호버링되어 있을 때는 커서 숨김 기능이 작동을 일시정지**하도록 안전 차단 장치를 구현했습니다.

## 4. 백그라운드 구동 시스템 (`run.bat`)
* 사용자가 바탕화면 등에서 `run.bat`를 더블클릭하면 다음과 같이 정밀하게 백그라운드 환경이 정비됩니다.
  1. 포트 충돌을 막기 위해 기존 5173 포트를 점유하고 있던 유령 Node 프로세스를 자동으로 종료(Free-up)합니다.
  2. 로컬 개발 서버(`npm run dev`)를 작업 표시줄의 **최소화(Minimized) 상태 창**으로 띄워 모니터를 어지럽히지 않습니다.
  3. 로컬 서버의 주소(`http://localhost:5173/`)를 크롬 앱 모드로 전체 화면 가깝게 띄운 후, **명령 프롬프트(CMD) 실행 창 본체는 스스로 종료(`exit`)되어 자동으로 사라집니다.**
